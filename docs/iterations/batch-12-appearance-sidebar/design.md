# Batch 12: Appearance 面板移至右侧栏 — 设计方案

## 设计目标

将 Appearance 设置从现有位置迁移到右侧栏，并由左侧 command bar 图标触发，保持设置即时生效。

## 设计原则

1. 入口明确: command bar 图标作为统一入口。
2. 面板一致: 右侧栏交互与现有侧栏模式保持一致。
3. 零行为回归: 外观设置项本身不改语义，仅改承载位置。
4. 即时应用: 设置变更实时作用于当前界面。

## 交互模型

### 打开与关闭

1. 点击 command bar Appearance 图标 -> 打开右侧 Appearance 面板。
2. 再次点击图标或点击面板关闭按钮 -> 关闭面板。
3. 面板打开状态需与其他右侧面板协调（避免重叠冲突）。

### 面板内容

沿用现有 Appearance 设置项：

- 主题/颜色
- 消息图标开关
- 用户图标选择

## 结构调整

1. 将 Appearance 面板 DOM 挂载到右侧栏容器体系。
2. 增加 command bar 图标及激活态管理。
3. 增加面板状态同步（打开、关闭、切换）。

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 与 AI Companion 右栏冲突 | 增加统一面板状态协调逻辑 |
| 迁移后样式错位 | 复用右栏容器样式并做移动端验证 |
| 设置即时应用失效 | 保留原设置应用链路，只迁移 UI 承载 |

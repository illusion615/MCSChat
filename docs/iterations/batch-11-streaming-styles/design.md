# Batch 11: Streaming 样式与速度优化 — 设计方案

## 设计目标

在保持现有消息正确性的前提下，提供可配置的流式呈现风格和速度，让用户按阅读偏好选择输出节奏。

## 设计原则

1. 体验优先: 样式变化不影响消息内容正确性。
2. 配置即生效: 调整后对新消息立即生效。
3. 可持久化: 用户偏好写入本地配置。
4. 可退化: 任意样式异常时可回退到即时显示。

## 样式策略设计

定义 streaming 渲染策略集合：

1. typewriter: 按字符推进。
2. word: 按词推进。
3. sentence: 按句推进。
4. fade: 分段淡入。
5. instant: 直接展示。

每个策略包含：

- 分段器（如何切片）
- 调度器（如何控制节奏）
- 收口器（如何结束与清理）

## 速度模型

统一速度参数为每秒更新频率或延时档位：

- slow
- normal
- fast
- ultra

不同样式可映射不同默认值，避免同速度下体验差异过大。

## 配置模型

新增配置项：

- streamingStyle
- streamingSpeed
- streamingPreviewEnabled

并写入本地存储，初始化时加载。

## UI 设计

在设置面板新增 Streaming 区域：

1. 样式选择器
2. 速度滑块或档位选择
3. 预览区域（可选）

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 动画过密造成卡顿 | 加入帧率保护与最小更新间隔 |
| 样式策略行为不一致 | 提供统一策略接口与收口规范 |
| 配置变更影响历史消息 | 仅对新消息生效，历史消息不重播 |

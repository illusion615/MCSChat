# 语音播放内容优化

## 问题描述

在使用Azure Speech Service播放消息内容时存在以下不良体验：

1. **Markdown字符被读出**：会将markdown中的`#`、`*`、`**`等格式字符也读出来
2. **引用标记被读出**：文字中的引用如`[1]`、`[2]`等也会被语音播报
3. **参考资料被读出**：内容末尾的"References"或"参考资料"部分无需语音播报
4. **超链接被读出**：URL和链接地址会被完整读出，听感体验差
5. **缺乏自然停顿**：章节标题和段落之间没有适当停顿，影响听觉理解

## 解决方案

### 1. 增强的文本清理功能（含智能停顿）

在 `speechEngine.js` 中重新设计了 `cleanTextForSpeech` 方法，在清理格式的同时保留必要的停顿：

#### Markdown格式清理（保留语义停顿）
- 移除粗体标记：`**文本**` → `文本`
- 移除斜体标记：`*文本*` → `文本`
- 移除代码标记：`` `代码` `` → `代码`
- 移除删除线：`~~文本~~` → `文本`
- **智能标题处理**：`## 标题` → `标题。` （添加句号创造停顿）
- **智能列表处理**：`- 项目` → `项目，` （添加逗号创造小停顿）
- 移除引用标记：`> 引用` → `引用`

#### 停顿优化策略
- **章节标题**：在标题后添加句号（。）创造明显停顿
- **列表项**：在列表项后添加逗号（，）创造轻微停顿  
- **段落分隔**：双换行转换为句号+空格（。 ）创造段落停顿
- **智能标点**：清理重复标点，确保停顿自然

#### 引用和链接清理
- 移除数字引用：`文本[1]` → `文本`
- 移除链接，保留显示文本：`[显示文本](url)` → `显示文本`
- 移除URL：自动删除http/https/ftp/www链接
- 移除邮箱地址

#### 参考资料清理
- 移除英文参考资料：`References:` 及其后续内容
- 移除中文参考资料：`参考资料:`、`相关链接:`、`资料来源:` 及其后续内容

### 2. Azure Speech Service集成

确保Azure Speech Service在两种模式下都应用增强的文本清理：

#### 高自然度模式 (naturalness > 0.7)
- 在`createSSML`方法中应用智能文本清理
- 生成增强的SSML标记，保留停顿效果

#### 标准模式 (naturalness ≤ 0.7)  
- 直接应用智能文本清理
- 包装在基础SSML结构中，保持停顿

### 3. Web Speech API兼容
更新Web Speech API的文本清理功能，确保两个语音引擎行为一致，都支持智能停顿。

## 测试用例

### 示例1：引用标记清理
**输入**：`请去系统设置菜单中关闭该选项[1]。这样可以提高系统性能[2]。`  
**输出**：`请去系统设置菜单中关闭该选项。这样可以提高系统性能。`

### 示例2：智能停顿处理
**输入**：
```
# 重要提醒

请注意以下几点：
- **重要**：务必备份数据
- 建议：定期更新系统

感谢您的配合。
```
**输出**：`重要提醒。 请注意以下几点： 重要：务必备份数据， 建议：定期更新系统， 感谢您的配合。`

**语音效果**：
- "重要提醒"后有明显停顿（句号）
- 列表项之间有轻微停顿（逗号）
- 段落间有适当停顿

### 示例3："Key Entities"类型独立标题停顿
**输入**：
```
Key Entities
• Artemis Program: NASA's initiative to return astronauts to the Moon.
• Space Launch System (SLS): Central rocket for Artemis, built from Space Shuttle parts.
```
**输出**：`Key Entities： Artemis Program: NASA's initiative to return astronauts to the Moon， Space Launch System (SLS): Central rocket for Artemis, built from Space Shuttle parts，`

**语音效果**：
- "Key Entities"后有明显停顿（冒号）
- 列表项之间有轻微停顿（逗号）
- 避免了标题直接连接到内容的问题

### 示例4：章节结构处理
**输入**：
```
## 1.1 基础概念

这是基础概念的介绍。

## 1.2 注意事项

- 第一点：记住备份
- 第二点：定期检查
```
**输出**：`1.1 基础概念。 这是基础概念的介绍。 1.2 注意事项。 第一点：记住备份， 第二点：定期检查，`

**语音效果**：
- 小节标题后有停顿
- 段落间有适当停顿
- 列表项间有轻微停顿

## 影响的文件

- `src/services/speechEngine.js`：主要修改文件
  - 重新设计了`cleanTextForSpeech`方法，加入智能停顿逻辑
  - 修改了`createSSML`方法以应用增强的文本清理
  - 确保Azure非SSML模式也应用智能停顿

## 测试文件

- `test-speech-cleanup.html`：用于验证文本清理和停顿功能的测试页面

## 使用说明

修改后的语音引擎会自动应用这些智能文本处理规则，包括：

1. **格式清理**：移除markdown格式符号、引用标记和参考资料
2. **智能停顿**：在标题、段落、列表项之间添加适当停顿
3. **自然播报**：提供更接近人工朗读的语音体验

用户在使用语音播放功能时，将获得更自然、更易理解的听觉体验，既没有格式干扰，又保留了内容的逻辑结构。

Speech stopping has been implemented in the following locations:

  • Application.sendMessage() - Stops speech when user clicks send or presses Enter
  • MessageRenderer.renderCompleteMessageDirect() - Stops speech when user messages are rendered
  • MessageRenderer.handleStreamingMessageDirect() - Stops speech during streaming user messages
  • MessageRenderer.simulateStreamingDirect() - Stops speech during simulated streaming user messages

ðŸ”§ How It Works

When a user sends a message:

  1. The sendMessage() method immediately calls aiCompanion.stopSpeaking()
  2. If the user message gets rendered, additional checks in the message renderer also stop any ongoing speech
  3. This ensures speech stops regardless of the message flow (direct send, programmatic, etc.)
